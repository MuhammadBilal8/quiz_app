Contributing
============

Thanks for your interest in contributing to this project! The following guidelines will help you make contributions that are easy to review and merge.

How to contribute

1. Fork the repository and clone your fork.
2. Create a feature branch: `git checkout -b feature/your-feature`.
3. Make changes in small, focused commits and run the app/tests locally.
4. Push your branch and open a Pull Request describing your changes.

Code style

- Keep code readable and follow existing project conventions.
- For Dart/Flutter, prefer the default formatter: `dart format .`.
- Add or update tests where applicable.

Testing

- Run unit and widget tests with:

```bash
flutter test
```

Reporting issues

- Open an issue with a clear title and steps to reproduce.
- Include environment details (OS, Flutter version, device/emulator).

Support

If you're not sure where to start, open an issue labeled `help wanted` or `good first issue` and we can point out a good first task.
